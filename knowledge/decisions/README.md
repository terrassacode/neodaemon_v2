# Decisions

## Purpose

`knowledge/decisions/` is reserved for stable decision records that should remain easy to find.

It may summarize accepted decisions after they are explicitly approved.

## Policy

This folder is documental only.

It does not change runtime behavior.

It does not replace `MEMORY.md`.

## Single Source of Truth

Each decision document must name the authoritative source, such as a merged PR, approved design, or explicit Albert decision.

Do not duplicate unresolved discussion as a decision.

## Creation rule

Do not create decision documents without an explicit decision.

No decision content is migrated in this seed.
