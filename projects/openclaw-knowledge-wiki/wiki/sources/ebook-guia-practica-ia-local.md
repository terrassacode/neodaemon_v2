## Resumen

Ebook introductorio sobre IA local: explica por qué ejecutar modelos en el propio ordenador puede mejorar privacidad, costes y disponibilidad offline, y presenta herramientas prácticas como Ollama y LM Studio para empezar a usar modelos locales.

## Datos confirmados

- El documento se titula “Guía práctica sobre IA local: qué es y cómo usarla”.
- La idea central es ejecutar modelos de IA en la máquina propia en vez de enviar datos a servicios cloud.
- Plantea tres ventajas principales:
  - privacidad: el código/datos no salen del ordenador;
  - costes: evita depender de cuotas o tokens de APIs externas;
  - modo offline: permite trabajar sin conexión.
- Presenta Ollama como herramienta de terminal para ejecutar modelos locales.
- Menciona modelos como Llama 3 y modelos especializados en código.
- Presenta LM Studio como opción con interfaz visual.
- Indica que Ollama y LM Studio pueden levantar un servidor local para conectar editores de código.
- Menciona Qwen Coder para tareas de programación.

## Inferencias

- El contenido encaja con la dirección estratégica de NeoDaemon/OpenClaw: sistema local, privado y con menor dependencia de servicios externos.
- La recomendación práctica para Albert sería priorizar herramientas locales simples antes de arquitecturas complejas.
- El ebook puede servir como fuente introductoria para una nota de “IA local para NeoDaemon”.
