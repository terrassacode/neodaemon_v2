# CORECLAW memory maintenance

## Fuente

- `raw/notes/CORECLAW-memory-soul-parte-3.md`

## Resumen

CORECLAW define mantenimiento de memoria como una práctica de selección: conservar lo que seguirá siendo útil y mover, resumir o descartar lo que sea táctico, sensible o caduco.

## Datos confirmados

- `memory/YYYY-MM-DD.md` captura el día.
- `MEMORY.md` conserva memoria curada y duradera.
- `memory/topics/*.md` guarda detalle persistente por tema o proyecto.
- El mantenimiento implica quitar ruido, fusionar duplicados, actualizar prioridades, mover detalle excesivo y mantener claridad.
- La memoria no debe intentar recordarlo todo.

## Inferencias

- La memoria útil requiere poda activa, no solo acumulación.
- Separar niveles reduce riesgo de exposición de información sensible y mejora recuperación.

## Dudas o límites

- Falta definir una cadencia concreta de mantenimiento.
- Falta decidir qué topic files privados convienen para Albert y NeoDaemon.
