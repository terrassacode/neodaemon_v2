# CORECLAW TOOLS reference map

## Fuente

- `raw/notes/CORECLAW-tools-parte-5.md`

## Resumen

`TOOLS.md` debe actuar como mapa de referencia local: rutas, IDs, endpoints, hosts, logs, CLIs, reglas específicas y preferencias técnicas necesarias para operar sin perder tiempo.

## Datos confirmados

- No debe contener secretos completos.
- Debe referenciar ubicaciones sensibles en vez de copiar contenido sensible.
- Puede describir canales, topics y comportamientos por superficie.
- Puede describir proyectos, CLIs, logs, bases locales, hosts y endpoints.
- Puede registrar reglas operativas propias de la instalación.
- Debe mantenerse corto y fácil de escanear.

## Inferencias

- Este mapa reduce dependencia de memoria frágil y búsquedas repetidas.
- Mantenerlo limpio evita mezclar detalles técnicos con memoria personal o reglas de personalidad.

## Dudas o límites

- Pendiente rellenar campos reales del entorno.
